## 简介

**Raccoon Agents** 是一个 Pi 扩展（Pi Extension），核心理念是：**用户只需要和浣熊讨论需求、创建 Issue，剩下的交给 Agent 们自动完成。**

它将你的开发工作流自动化为以下闭环：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  需求讨论   │ ──▶ │  创建 Issue │ ──▶ │  Agent 开发  │ ──▶ │   PR 合并   │
│ (用户+浣熊) │     │  (自动转换) │     │ (多模型编排) │     │ (自动提交)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ▼
                                       ┌─────────────┐
                                       │  任务拆分   │
                                       │ (智能分解)  │
                                       └─────────────┘
```

## 核心理念

| 传统开发 | 浣熊特工队 |
|----------|-----------|
| 需求评审 → 手动创建任务 → 人工分配 → 逐一开发 → 手动提 PR → 人工 CR | **用户开口说需求 → 自动完成一切** |
| 多人协作、进度同步、上下文切换 | 多模型 Agent 并行编排、各司其职 |
| 模型能力单一 | 多模型协作，各取所长 |

> 💡 **你只需做两件事**：和浣熊讨论需求、确认/创建 Issue。写代码、拆任务、跑测试、提 PR、合并分支……全部自动完成。

## 智能模型调度

浣熊特工队的核心优势之一：**不是固定分配模型，而是根据任务阶段和实时表现动态升降模型档位**，在效果与成本之间取得最佳平衡。

### 调度策略

| 阶段 | 模型档位 | 策略说明 |
|------|---------|---------|
| **需求理解** | 高档 / 中档 | 根据需求复杂度自动判断。涉及系统架构、跨模块协调、模糊需求澄清时使用高档模型；需求明确、范围有限时使用中档。 |
| **任务拆分** | 高档 / 中档 | 复杂任务的拆分（如需要识别隐式依赖、设计并发执行策略）使用高档模型；简单拆分（如按模块划分）使用中档。 |
| **任务执行** | 中档 / 低档 | **优先使用低档模型**。常规编码、文档生成、简单测试编写走低档；遇到复杂算法、性能优化、疑难 Bug 修复时升级中档。 |
| **代码审核** | 中档 | 统一由中档模型执行审核，检查代码规范、逻辑正确性、边界条件、安全漏洞。 |
| **故障救援** | 高档介入 | 当任务多次执行不达标（审核不通过、测试失败、输出质量低于阈值），浣熊自动判断并召唤高档模型介入指导，分析问题根因并给出修复方案。 |

### 自动升降级流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              智能模型调度引擎                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   需求输入 ──▶ [复杂度评估] ──▶ 高档模型(复杂)                              │
│                        └──────▶ 中档模型(一般)                              │
│                                                                             │
│       ▼                                                                     │
│   任务拆分 ──▶ [依赖分析] ────▶ 高档模型(高耦合)                            │
│                        └──────▶ 中档模型(低耦合)                            │
│                                                                             │
│       ▼                                                                     │
│   任务执行 ──▶ [优先低档] ────────────────────────────┐                     │
│                        │                             │                     │
│                        ▼                             ▼                     │
│                  [中档升级] 复杂/疑难          [低档执行] 常规任务           │
│                                                                             │
│       ▼                                                                     │
│   代码审核 ──▶ [中档模型] ──▶ 通过 ────────────────▶ PR 提交               │
│                        │                                                   │
│                        ▼ 不通过                                            │
│                   [失败计数]                                               │
│                        │                                                   │
│                        ▼ 连续 N 次失败                                     │
│                   [高档模型介入] ──▶ 根因分析 + 修复指导                     │
│                        │                                                   │
│                        ▼                                                   │
│                   重新分配执行模型                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 设计原则

- **能省则省**：90% 的日常编码任务由中低档模型完成，只在真正需要推理能力的地方启用高档模型。
- **自动兜底**：不依赖人工判断何时升级模型，浣熊根据审核结果和失败次数自动决策。
- **闭环学习**：高档模型介入后给出的分析和修复方案会被记录，用于优化后续同类任务的拆分策略和执行模型选择。

## 安装

Raccoon Agents 以 [Pi Package](https://pi.dev) 形式分发。

### 方式一：全局安装（推荐）

安装到 Pi 全局，所有项目均可使用：

```bash
# 从 Git 安装
pi install git:github.com/<user>/raccoon-agents

# 从 npm 安装（若已发布）
pi install npm:raccoon-agents
```

### 方式二：克隆后本地运行（开发调试）

先克隆仓库到本地：

```bash
git clone <repo-url> raccoon-agents
cd raccoon-agents
npm install
```

然后在克隆目录中直接启动：

```bash
# 使用 package.json 中的 start 脚本
npm start

# 或手动指定当前目录作为扩展路径
pi -e .
```

### 方式三：项目级本地安装

如果你想把本扩展安装到某个特定项目（仅该项目可用），先按「方式二」克隆到本地，然后进入目标项目执行：

```bash
cd /path/to/your-project
pi install -l /path/to/raccoon-agents
```

`-l`（`--local`）会将扩展安装到当前项目的 `.pi/` 目录下，不影响全局配置。

### 验证安装

启动 Pi 后，若顶部出现浣熊特工队 ASCII 艺术标题，即表示扩展已加载成功。

## 使用方式

### 1. 需求讨论

像和同事一样和浣熊讨论你的需求：

```
用户：我想加一个用户登录功能，支持邮箱和密码，还要能记住我
浣熊：收到！我来分析一下需求……需要以下功能点：
      1. 登录页面 UI
      2. 后端登录 API
      3. 密码加密存储
      4. Session / Token 管理
      5. "记住我" 持久化
      是否创建 Issue 开始自动开发？
用户：好，创建吧
```

### 2. 自动创建 Issue

浣熊自动将需求转换为 Git Issue，包含完整描述、验收标准和标签。

### 3. Agent 自动开发（无需干预）

Issue 创建后，浣熊自动：
- 拆分任务到多个子任务
- 为每个子任务分配最合适的模型
- 并行启动多个 Agent 进行开发
- 自动运行测试
- 自动提交 PR

### 4. 自动合并

PR 通过 CI 后自动合并，用户收到完成通知。

## 配置

Raccoon Agents 通过 `package.json` 中的 `pi` 字段向 Pi 注册扩展入口：

```json
{
  "keywords": ["pi-package"],
  "pi": {
    "extensions": [
      "./extensions/index.ts"
    ]
  }
}
```

### 多模型配置（即将支持）

未来将支持在 `.pi/settings.json` 中配置不同任务的模型偏好：

```json
{
  "raccoonAgents": {
    "models": {
      "architecture": "claude-sonnet-4",
      "frontend": "openai/gpt-4o",
      "backend": "claude-sonnet-4",
      "docs": "google/gemini-2.5-pro",
      "test": "openai/gpt-4o-mini"
    }
  }
}
```
